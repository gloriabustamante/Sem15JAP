División (Dividir el problema)
El primer paso es dividir el problema original en subproblemas. La idea es identificar partes más pequeñas y específicas del problema más grande, cada una de las cuales pueda resolverse de manera independiente.
Ejemplo: Si el problema es ordenar una lista de números, el problema puede dividirse en subproblemas que ordenen partes más pequeñas de la lista. Esto no solo hace que el problema sea más manejable, sino que también permite utilizar diferentes enfoques para resolver cada subproblema.
Adicionalmente, al dividir el problema, se debe considerar si los subproblemas son lo suficientemente similares para aplicar la misma estrategia de solución, ya que esto optimiza el proceso.

Conquista (Resolver los subproblemas)
Una vez que los subproblemas están claramente definidos, se resuelven recursivamente (o iterativamente). La solución para cada subproblema es más simple que la del problema original.
Es crucial abordar cada subproblema de forma independiente, asegurando que cada uno se resuelva correctamente. Este paso puede involucrar aplicar la misma técnica de división a los subproblemas hasta llegar a instancias que sean triviales de resolver, como un solo elemento en una lista.
Por ejemplo, en el caso del algoritmo de ordenación por mezcla, se continúa dividiendo la lista hasta que se obtienen listas de un solo elemento, que son inherentemente ordenadas.

Combinación (Combinar las soluciones)
Después de resolver los subproblemas, las soluciones individuales se combinan para formar la solución del problema original.
Este paso es fundamental, ya que la combinación de las soluciones debe hacerse de manera que mantenga la integridad de la solución final. Es posible que se requiera lógica adicional para unir las soluciones, especialmente si hay dependencias entre los subproblemas.
En el caso de un algoritmo como el de ordenación por mezcla, esto implicaría fusionar las listas ordenadas resultantes en una lista única y ordenada, asegurando que todos los elementos estén en el orden correcto.

Recurrencia
Repetir el proceso recursivamente para cada subproblema hasta que los subproblemas sean lo suficientemente pequeños como para resolverse directamente.
Este paso destaca la naturaleza recursiva de muchos algoritmos que aplican la estrategia de Divide y Vencerás. Es importante establecer condiciones de parada que eviten que la recursión se vuelva infinita.
Por ejemplo, si se está implementando una función recursiva para calcular el factorial de un número, la condición de parada sería cuando el número es igual a 1. Esto garantiza que eventualmente se llegue a un caso base que se puede resolver sin más división.
